    • Basics

    Just like URL's of the world wide web, linkspace looks a lot like a filesystem.
    To understand the difference, we must first look at merging two sets.

    /image/Broken.jpg [image data] /thread/Coffee machine broke!/msg fix pls? image/Broken.jpg
    +
    /thread/Tabs or spaces/msg Are we still doing this?
    =
    /image/Broken.jpg [image data] /thread/Coffee machine broke!/msg fix pls? image/Broken.jpg /thread/Tabs or spaces/msg Are we still doing this?

    A web-servers is software that serve such a file/folder tree, receives new entries, and does some additional processing1. Your browser requests some of the data in the tree and uses it draw a interface to make it easy to interact with. The webserver does not care about those graphics. It only knows about data. What happens when two people want to write to the same location? A server would have multiple options. After all, it is the only 'real' copy we can point to. One option is to only keep one entry, or add a timestamp to the end signaling the date it received the message. Without a single host being the 'real' copy, these options don't work. But having a single host is slow, creates a single point of failure, and with that much control by one organization has profound consequences.

    In linkspace two entries can always overlap

    [HASH_0] /thread/Tabs or spaces/msg Are we still doing this?
    +
    [HASH_1] /thread/Tabs or spaces/msg Why not U+3164?
    +
    [HASH_2] /thread/Tabs or spaces/msg Get a life
    =
    [HASH_0] /thread/Tabs or spaces/msg Are we still doing this? [HASH_1] /thread/Tabs or spaces/msg Why not U+3164? [HASH_2] /thread/Tabs or spaces/msg Get a life

    The hash is a unique number for every entry. It depends in part on the content and the path.
    This allows us to merge two trees.
    What is more, we also get the option to pick if we want to refer to things by a path "image/Broken.jpg" or by their hash.

    [HASH_3] /image/Broken.jpg [image data] [HASH_4] /thread/Coffee machine broke!/msg fix pls? [HASH_3]image/Broken.jpg
    +
    [HASH_5] /thread/Coffee machine broke!/msg Hey [HASH_4]! Isn't this [HASH_3] image from 2015?
    =
    [HASH_3] /image/Broken.jpg [image data] [HASH_4] /thread/Coffee machine broke!/msg fix pls? image/Broken.jpg [HASH_5] /thread/Coffee machine broke!/msg Hey [HASH_4]! Isn't this [HASH_3] image from 2015?

    Linkspace goes further. Each entry is set inside a Domain and a Group. A domain name is chosen by a developer creating a (graphical) interface. The domain name signals what app to use, and thus how the developer organized entries. The group is set by the user. It exits because we say it exists. Finally, each entry has a creation date and can be signed.

    [HASH_3] message_board
    [#:example:com]     /image/Broken.jpg
    2015/01/02     [image data]
    [@:alice:sales:com]     [HASH_4] message_board
    [#:example:com]     /thread/Coffee machine broke!/msg
    2023/03/02     fix pls? image/Broken.jpg
    [@:alice:sales:com]     [HASH_5] message_board
    [#:example:com]     /thread/Coffee machine broke!/msg
    2023/03/03     Hey [HASH_3]! Isn't this [HASH_2] image from 2015?
    [@:bob:maintainance:com]
